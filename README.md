# СВОИ — Tinder для совместной аренды жилья

Рабочий hackathon MVP платформы, которая подбирает квартиру и совместимых соседей. Основная демо-локация — Краснодар.

## Что реализовано

- адаптивный современный интерфейс с анимациями и glassmorphism;
- 128 демонстрационных объявлений с фотографиями, адресами, расстоянием, ценой и свободными местами;
- 50 профилей потенциальных соседей;
- ярлыки привычек и отдельные red flags для пользователей и квартир;
- ранжирование по бюджету, расстоянию, привычкам, ограничениям и количеству свободных мест;
- Tinder-механика поиска соседа;
- взаимный мэтч, чат и подтверждение готовности жить вместе;
- постоянное лобби арендаторов;
- автоматический пересчёт квартир под общий бюджет и состав лобби;
- общий чат с владельцем и уже проживающим жильцом;
- демонстрационные ответы ботов на вопросы о курении, питомцах, цене и залоге;
- форма размещения квартиры арендодателем;
- моковые проверки через Госуслуги, Сбер ID и паспорт;
- сохранение мэтчей, лобби, профиля и чатов в `localStorage`.

## Архитектура

Проект сделан как автономный SPA на HTML, CSS и JavaScript без backend-зависимостей. Это уменьшает риск проблем на защите: Nginx сразу отдаёт готовые файлы, а Docker-образ собирается за несколько секунд.

Серверная часть:

- Docker;
- Nginx;
- Docker Compose;
- GitHub Actions + SSH auto-deploy.

## Быстрый локальный запуск

```bash
python3 -m http.server 5173
```

Открыть: `http://localhost:5173`

Можно также просто открыть `index.html` в браузере, но HTTP-сервер предпочтительнее.

## Мгновенный запуск на Ubuntu-сервере

Требования: Ubuntu 22.04/24.04, открытый TCP-порт `80`, SSH-доступ и пользователь с `sudo`.

Выполните на сервере одним блоком:

```bash
sudo apt-get update && sudo apt-get install -y git ca-certificates curl && \
rm -rf /tmp/svoi-bootstrap && \
git clone https://github.com/Vatnik12/Hakaton.git /tmp/svoi-bootstrap && \
cd /tmp/svoi-bootstrap && \
sudo bash deploy/setup-server.sh
```

Скрипт автоматически:

1. установит Docker и Docker Compose;
2. создаст пользователя `deploy`;
3. клонирует проект в `/opt/svoi`;
4. запустит контейнер на порту `80`;
5. выведет ссылку вида `http://123.123.123.123`;
6. создаст SSH-ключ для GitHub Actions.

После запуска сайт доступен сразу по IP без домена:

```bash
SERVER_IP=$(curl -4 -s https://api.ipify.org)
echo "Открыть сайт: http://$SERVER_IP"
```

Если включён UFW:

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw --force enable
```

## Автодеплой из GitHub

После выполнения `deploy/setup-server.sh` скрипт покажет IP, имя пользователя и приватный SSH-ключ.

Откройте:

`Settings -> Secrets and variables -> Actions -> New repository secret`

Добавьте:

| Secret | Значение |
|---|---|
| `SERVER_HOST` | IP сервера |
| `SERVER_USER` | `deploy` |
| `SSH_PRIVATE_KEY` | приватный ключ целиком |

После этого каждый push в `main` автоматически обновляет `/opt/svoi` и пересобирает контейнер. Пока секреты не добавлены, workflow корректно пропускает деплой без ошибки.

Workflow: `.github/workflows/deploy.yml`.

## Ручное обновление

```bash
sudo -u deploy /usr/local/bin/deploy-svoi
```

## Полезные команды

```bash
# Статус
docker ps --filter name=svoi-web

# Логи
docker logs -f svoi-web

# Пересобрать
docker compose -f /opt/svoi/docker-compose.yml up -d --build

# Остановить
docker compose -f /opt/svoi/docker-compose.yml down

# Узнать публичный IP
curl -4 https://api.ipify.org
```

## Сценарий демонстрации

1. Нажать «Найти своих».
2. Открыть профиль и выбрать свои ярлыки и стоп-факторы.
3. Перейти в «Найти соседа» и поставить лайк.
4. В чате нажать «Я готов» — создастся общее лобби.
5. Вернуться к квартирам: бюджет и минимальное количество мест пересчитаются под команду.
6. Открыть квартиру и отправить заявку.
7. В общем чате спросить про питомцев, курение, цену или залог.
8. Показать верификацию и форму арендодателя.

## Ограничения MVP

Это демонстрационный продукт. Для production нужны backend, база данных, реальная авторизация, WebSocket-чаты, серверная модерация, безопасная загрузка файлов и официальные интеграции верификации.
