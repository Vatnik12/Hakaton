# СВОИ — Tinder для совместной аренды жилья

Рабочий hackathon MVP платформы, которая подбирает не только квартиру, но и совместимых соседей. Основная демо-локация — Краснодар.

## Что реализовано

- премиальный адаптивный интерфейс для мобильных и десктопных браузеров;
- 128 демонстрационных объявлений с фотографиями, адресами, расстоянием и свободными местами;
- 50 профилей потенциальных соседей;
- ярлыки привычек и отдельные red flags для пользователей и квартир;
- объяснимое ранжирование по бюджету, расстоянию, привычкам, ограничениям и количеству свободных мест;
- Tinder-механика поиска соседа;
- взаимный мэтч, чат и подтверждение готовности жить вместе;
- постоянное лобби из нескольких арендаторов;
- автоматический пересчёт рейтинга квартир под общий бюджет и объединённые признаки лобби;
- общие чаты с владельцем и уже проживающими жильцами;
- демонстрационные ответы ботов на вопросы о курении, питомцах, цене, уборке и просмотре;
- размещение нового объявления арендодателем;
- моковые проверки через Госуслуги, Сбер ID и паспорт;
- состояние сохраняется в `localStorage`, поэтому мэтчи, лобби и чаты не пропадают после обновления страницы.

## Стек

- React + TypeScript;
- Vite;
- CSS animations / glassmorphism без тяжёлых UI-зависимостей;
- Docker multi-stage build;
- Nginx;
- GitHub Actions + SSH auto-deploy.

## Мгновенный запуск на Ubuntu-сервере

Требования: чистый Ubuntu 22.04/24.04, открытый TCP-порт `80`, доступ по SSH и пользователь с `sudo`.

Скопируйте и выполните **одним блоком** на сервере:

```bash
sudo apt-get update && sudo apt-get install -y git ca-certificates curl && \
git clone https://github.com/Vatnik12/Hakaton.git /tmp/svoi-bootstrap && \
cd /tmp/svoi-bootstrap && \
sudo bash deploy/setup-server.sh
```

Скрипт автоматически:

1. установит Docker и Docker Compose;
2. создаст пользователя `deploy`;
3. клонирует проект в `/opt/svoi`;
4. соберёт и запустит контейнер на порту `80`;
5. выведет готовую ссылку вида `http://123.123.123.123`;
6. создаст SSH-ключ для GitHub Actions и покажет, что добавить в Secrets.

После запуска сайт доступен **сразу по IP, без домена**:

```bash
SERVER_IP=$(curl -4 -s https://api.ipify.org)
echo "Открыть сайт: http://$SERVER_IP"
```

Если сервер использует UFW:

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw --force enable
```

## Настройка автодеплоя

После выполнения `deploy/setup-server.sh` скрипт выведет приватный ключ и значения переменных.

В репозитории откройте:

`Settings -> Secrets and variables -> Actions -> New repository secret`

Добавьте три секрета:

| Secret | Значение |
|---|---|
| `SERVER_HOST` | IP сервера |
| `SERVER_USER` | `deploy` |
| `SSH_PRIVATE_KEY` | приватный ключ целиком, включая BEGIN/END |

После этого каждый push в ветку `main` автоматически выполнит на сервере:

```bash
cd /opt/svoi
git fetch origin main
git reset --hard origin/main
docker compose up -d --build --remove-orphans
```

Workflow находится в `.github/workflows/deploy.yml`.

## Ручное обновление на сервере

```bash
sudo -u deploy /usr/local/bin/deploy-svoi
```

Или из каталога проекта:

```bash
cd /opt/svoi
sudo -u deploy bash deploy/deploy.sh
```

## Полезные команды сервера

```bash
# Статус
docker ps --filter name=svoi-web

# Логи
docker logs -f svoi-web

# Пересобрать
docker compose -f /opt/svoi/docker-compose.yml up -d --build

# Остановить
docker compose -f /opt/svoi/docker-compose.yml down

# Узнать публичный IP
curl -4 https://api.ipify.org
```

## Демонстрационный сценарий для защиты

1. Нажать «Найти своих».
2. Открыть профиль и выбрать свои признаки и стоп-факторы.
3. Перейти в «Найти соседа», поставить лайк и открыть взаимный мэтч.
4. В чате нажать «Я готов» — второй участник подтвердит автоматически, после чего появится лобби.
5. Вернуться к квартирам: рейтинг, допустимый бюджет и минимальное число мест пересчитаются под команду.
6. Открыть квартиру и отправить заявку.
7. В общем чате задать вопросы про питомцев, курение, залог, уборку или просмотр.
8. Показать моковую верификацию и форму размещения квартиры.

## Важное для production

Это hackathon MVP. Для реального продукта следует заменить `localStorage` на backend и базу данных, добавить реальную авторизацию, WebSocket-чаты, серверную модерацию, безопасную загрузку файлов и официальные интеграции верификации.
