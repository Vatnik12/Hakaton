# Гнездо — сервис совместной аренды жилья

Рабочий hackathon MVP, который помогает найти не только квартиру, но и совместимых соседей. Основная демо-локация — Краснодар.

## Что реализовано

- новый фирменный стиль «Гнездо» с природной палитрой, птицами, ветвями и скролл-анимациями;
- адаптивный интерфейс для мобильных устройств и десктопа;
- 128 демонстрационных объявлений с фотографиями, адресами, расстоянием, ценой и свободными местами;
- 50 профилей потенциальных соседей;
- ярлыки привычек и отдельные red flags для пользователей и квартир;
- ранжирование по бюджету, расстоянию, привычкам, ограничениям и количеству свободных мест;
- Tinder-механика поиска соседа;
- взаимный мэтч, чат и подтверждение готовности жить вместе;
- постоянное «гнездо» арендаторов — общее лобби для совместного поиска;
- автоматический пересчёт квартир под общий бюджет и состав команды;
- общий чат с владельцем и уже проживающим жильцом;
- демонстрационные ответы ботов на вопросы о курении, питомцах, цене и залоге;
- форма размещения квартиры арендодателем;
- моковые проверки через Госуслуги, Сбер ID и паспорт;
- сохранение мэтчей, команды, профиля и чатов в `localStorage`.

## Архитектура

Проект сделан как автономный SPA без обязательного backend. Его можно сразу отдавать через Nginx или любой статический хостинг.

Основные файлы:

```text
index.html
app.js
brand.js
style.css
```

## Локальный запуск

```bash
python3 -m http.server 8080
```

Открыть:

```text
http://localhost:8080
```

## Запуск на Ubuntu-сервере

Выполните одним блоком:

```bash
sudo apt-get update && sudo apt-get install -y git ca-certificates curl && \
rm -rf /tmp/gnezdo-bootstrap && \
git clone https://github.com/Vatnik12/Hakaton.git /tmp/gnezdo-bootstrap && \
cd /tmp/gnezdo-bootstrap && \
sudo bash deploy/setup-server.sh
```

После установки сайт будет доступен по IP сервера без домена:

```bash
echo "http://$(curl -4 -s https://api.ipify.org)"
```

## Автодеплой

В GitHub откройте:

```text
Settings → Secrets and variables → Actions
```

Добавьте секреты, которые покажет `deploy/setup-server.sh`:

```text
SERVER_HOST
SERVER_USER
SSH_PRIVATE_KEY
```

После этого каждый push в `main` автоматически обновляет сайт на сервере через workflow `.github/workflows/deploy.yml`.

## Ручное обновление

```bash
sudo -u deploy /usr/local/bin/deploy-svoi
```

или:

```bash
cd /opt/svoi
sudo -u deploy bash deploy/deploy.sh
```

## Демонстрационный сценарий

1. Открыть «Гнездо».
2. Настроить профиль и стоп-факторы.
3. Перейти к поиску соседа.
4. Получить мэтч и открыть чат.
5. Подтвердить совместный поиск.
6. Получить пересчитанную выдачу квартир для всей команды.
7. Откликнуться на квартиру.
8. Открыть общий чат с владельцем и текущим жильцом.
9. Показать верификацию и кабинет арендодателя.

## Production-заметки

Для полноценного запуска продукта потребуется заменить `localStorage` на backend и базу данных, добавить авторизацию, WebSocket-чаты, серверную модерацию, безопасную загрузку файлов и реальные интеграции верификации.
